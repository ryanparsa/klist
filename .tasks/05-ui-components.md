# Task 05 — UI Components & Visual Polish

**Touches:** `src/`
**Goal:** Adopt shadcn/ui components and improve the visual quality of the checklist UI.

---

This is **klist** — a static Vite + React 19 app with Tailwind CSS v4 and shadcn/ui conventions. No backend. It loads `public/checklist.json` (generated by the builder — do not modify it) and renders a Kubernetes operational checklist.

**Stack:** Vite 6, React 19, Tailwind v4 (`@tailwindcss/vite`), `lucide-react`, `react-markdown` + `remark-gfm`, `react-syntax-highlighter`, `mermaid`.

**Source layout:**
```
src/
  App.jsx                        # root — layout, filtering, search
  main.jsx
  index.css
  lib/utils.js                   # cn() helper
  hooks/
    useChecklist.js              # fetches checklist.json
    useUrlState.js               # active configs/tags in query string
    useItemStates.js             # tri-state (pass / n-a / unchecked) per item
    usePriorityResolution.js     # resolves required/suggested from active configs
  components/
    Sidebar.jsx                  # config + tag filters
    ChecklistSection.jsx         # one 4C section (Cloud/Cluster/Container/Code)
    ChecklistItem.jsx            # individual item row with expand/collapse
    ScoreBar.jsx                 # per-section progress bar
    ExportImport.jsx             # JSON session export/import
    Markdown.jsx                 # renders description markdown
    MermaidDiagram.jsx           # renders mermaid fenced blocks
```

---

## Your task

1. **shadcn/ui adoption** — replace raw `<input>`, `<button>`, and layout primitives with proper shadcn/ui components (`Button`, `Input`, `Badge`, `Sheet` for the mobile sidebar, `ScrollArea`, etc.) where it reduces boilerplate and improves consistency. Install any missing shadcn components with `npx shadcn@latest add <component>`.

2. **ChecklistItem polish** — the expand/collapse item detail (description, mitigations, tools, references) should feel polished:
   - Smooth animate-in on expand
   - Clearly separated sections (description / mitigations / tools / references)
   - Code blocks syntax-highlighted via `react-syntax-highlighter`
   - Mermaid diagrams rendered inline via `MermaidDiagram.jsx`

3. **Sidebar visual clarity** — configs and tags are currently toggles; improve the active state so it is visually obvious. Group configs by type with a small section label:
   - **Baseline**
   - **Provider** (eks, aks, gke, self-managed)
   - **Architecture** (ha, multi-cluster, production)
   - **Compliance** (hipaa, pci-dss, soc2, cis-benchmark, owasp-k8s-top10)

4. **ScoreBar** — show item counts alongside the bar (e.g. "4 / 12 passed").

5. **Search** — the search filter in `App.jsx` should match on tags in addition to title and description.

Do **not** change the YAML data format, the builder, or `public/checklist.json`.
